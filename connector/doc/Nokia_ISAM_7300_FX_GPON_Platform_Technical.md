# Nokia ISAM 7300 FX GPON Platform

## About

The **Nokia ISAM 7300 FX GPON Platform** connector uses an SNMP connection to communicate with Nokia ISAM 7300 FX devices. The collected data is centralized in DataMiner and used by DataMiner EPM for aggregation and topology workflows.

## Configuration

### Connections

#### SNMP Main Connection

This connector uses Simple Network Management Protocol (SNMP) and requires the following input during element creation:

- **IP address/host**: Polling IP address or URL of the destination.
- **IP port**: IP port of the destination.
- **Get community string**: Community string used for read operations (default: *public*).
- **Set community string**: Community string used for write operations (default: *private*).

### Initialization

The connector uses custom properties to configure the Network, Market, and Hub of the OLT. To link views with EPM data cards and enable full EPM functionality, configure these properties.

The EPM solution uses a file-based exchange mechanism for topology communication. After element creation, configure these settings on the **Configuration** page:

- **Entity Export/Import Settings**:
  - **Export State** and **Import State**: Enable or disable export/import.
  - **Export Directory**, **Entity Import Directory**, and **ONT Import Directory**: Paths for exported and imported files.
  - **Entity Export Directory Type**, **Entity Import Directory Type**, and **ONT Import Directory Type**: Select *Local* or *Remote*. For remote paths, configure system credentials and ensure the paths are accessible.
- **System Credentials**:
  - **System Username**: Account with access to the remote directory. If no domain is specified, the DMA location domain is used.
  - **System Password**: Password for remote directory access.

### Redundancy

No redundancy is defined.

## How to Use - Range 1.0.0.x

The OLT connectors act as links in the EPM GPON solution chain. They represent the lower GPON topology layer, where ONT information is retrieved and then processed by upper layers. Some ONT KPIs can come from external sources (for example, Kafka), so communication health with these sources is important.

Set the **Reset ONT Interval** parameter (on the **Configuration** page) according to the ONT data retrieval speed.

After initial setup, the connector can run without additional configuration. The following actions and intervals are available on the **Configuration** page:

- **Entity Export Settings/Apply**: Exports topology configuration files for Skyline EPM Solution processing.
- **Entity Import Settings/Apply**: Imports provisioning files generated by Skyline EPM Solution.
- **SNMP Fast Interval**: Polling interval for OLT status data (default: 15 minutes).
- **SNMP Slow Interval**: Polling interval for OLT configuration data (default: 4 hours).
- **Virtual Interval**: Topology synchronization interval with EPM (default: 2 hours).
- **ONT Interval**: Interval for requesting ONT state data (for example from Kafka; default: 15 minutes).

Performance can vary depending on the number of updates received.

## How to Use - Range 1.0.1.x

In this range, the OLT is no longer responsible for retrieving information from external sources (for example, Kafka). As a result, the OLT Interval parameter was removed.

Passive logic was also updated: split information is now distributed over three tables to improve topology segmentation:

- **Split Route Overview**
- **Split Distribution Overview**
- **Split FAT Overview**

## How to Use - Range 1.0.2.x

In this range, the OLT can retrieve information through either file import (for example via remote connection) or SNMP polling. This includes additional EPM Registration Configuration that allows import of an ONT registration file.

This mechanism helps avoid duplicate polling across environments, which can otherwise negatively impact Nokia device performance.

By default, **SNMP Information Polling Method** is set to *File Import*. For correct operation, update EPM Import Settings:

- Import Directory path
- Import Directory type
- System credentials

## Notes

This connector requires specific Automation scripts for communication with auxiliary connectors, including Skyline EPM Platform GPON WM and DataMiner EPM Solution.

Polling performance may vary with larger devices or data sets.
