# Telenor EPM MAM Consumer

## About

In the Telenor DMS, Media Asset Management (MAM) events are published to RabbitMQ queues. The **Telenor EPM MAM Consumer** connector retrieves those messages, forwards them to the matching **Telenor EPM Collector** elements, and offloads them to OpenSearch through the **Telenor EPM MAM Offloader** element. It relies on the provisioning files generated by the **Telenor EPM Manager** element to know which collectors are available and where each message should go.

## Configuration

### Connections

#### Virtual Connection - Main

This connector uses a virtual connection and does not require any input during element creation.

### Initialization

Once the element has been created, configure the following before the connector can start processing messages:

- **Sources page**: Add the RabbitMQ servers to read from. Each source requires a name, IP address/host, port, username, password, virtual host, and SSL setting, and can be connected or disconnected from the table's right-click menu. The queue name is shared by all sources and is set on the **General** page, together with the prefetch count.
- **Collectors page**: Set the **provisioning folder** path and its access credentials (user, password, domain) so the connector can read the provisioning files created by the Telenor EPM Manager. Use the **Provision** button to read those files and populate the Collectors table. The forwarding period can also be configured here.
- **Offload pages**: On each offload page, enable or disable offloading to OpenSearch for that event type, and optionally copy the parsed messages to a local debug table. Cleanup settings (maximum age and maximum number of rows) control retention of the debug tables.
- **Log page**: Set the log level and configure the devices to trace. Use the devices table's right-click menu to add or remove devices for which received messages should be logged.

## How to Use

The connector reads MAM events from the configured RabbitMQ sources over the AMQP protocol. Because RabbitMQ is used as the transport, **no data traffic is visible in the Stream Viewer**.

After startup, the connector provisions the collectors and connects to all configured sources. Retrieved messages are then handled in two ways:

- **Forwarding to collectors**: Messages are routed to the matching Telenor EPM Collector elements based on the provisioning files, using DataMiner parameter sets. The **Collectors** page lists all detected collectors and their current state.
- **Offloading to OpenSearch**: When offloading is enabled for an event type, the messages are sent to the **Telenor EPM MAM Offloader** element, which writes them to OpenSearch. Offloading can be toggled independently for each of the supported event types: Error, Activity, Diagnostics, ContentWise, AppSystemEvent, ChangeSetting, PlayerStats, ExecuteAction, ScreenView, OpenLandingPage, and Playback.

For troubleshooting, the **Log** page lets you enable custom logging for specific devices. The messages received for those devices are written to log files stored in the folder `C:\Skyline DataMiner\Telenor\Elements\<ElementName>\Logging`.
