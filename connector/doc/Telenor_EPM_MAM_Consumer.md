# Telenor EPM MAM Consumer

## About

In the Telenor DMS, MAM events are published to **RabbitMQ** queues across multiple data centers. The **Telenor EPM MAM Consumer** connector consumes those messages and routes them to where they are needed: it **forwards** them to the matching **Telenor EPM Collector** elements and **Telenor EPM MAM Offloader** element for long-term storage and analysis.

The connector stays aware of the available collectors by reading the provisioning files generated by the **Telenor EPM Manager** element, so message routing keeps working as the environment changes.

## Key Features

- **Multi-data-center consumption**: Connect to several RabbitMQ servers at once and read MAM events from a shared queue across all of them.
- **Automatic collector routing**: Detect the correct destination for every message based on the provisioning files created by the Telenor EPM Manager, with no manual mapping.
- **Selective OpenSearch offloading**: Enable or disable offloading per event type, so only the data you need is stored downstream.
- **Broad MAM event coverage**: Parse and handle a wide range of event types, including error, activity, diagnostics, ContentWise, playback, player statistics, and user-interaction events.
- **Targeted device logging**: Capture the messages received for specific devices in dedicated log files to speed up troubleshooting.

## Use Cases

### Consolidating MAM events from multiple data centers

**Challenge**: MAM events are spread across RabbitMQ queues in several data centers, making them hard to collect and analyze consistently.

**Solution**: The connector consumes from all configured RabbitMQ sources simultaneously and channels every event through a single, uniform pipeline.

**Benefit**: A single element provides one consistent entry point for all MAM traffic, reducing operational complexity.

### Keeping message routing in sync with a changing environment

**Challenge**: Collectors are added, removed, or reassigned over time, and static routing quickly becomes outdated.

**Solution**: The connector reads the provisioning files produced by the Telenor EPM Manager to keep its collector list and routing up to date automatically.

**Benefit**: Messages keep reaching the right collector without manual reconfiguration whenever the topology changes.

## Technical Reference

### Prerequisites

- A **DataMiner Agent** running version **10.3.0** (build 12752) or higher.
- Access to one or more **RabbitMQ** servers exposing the MAM event queue.
- A **Telenor EPM Manager** element that generates the provisioning files, and access to the folder where those files are stored.
- **Telenor EPM Collector** elements to forward messages to.
- A **Telenor EPM MAM Offloader** element to offload messages to OpenSearch.
