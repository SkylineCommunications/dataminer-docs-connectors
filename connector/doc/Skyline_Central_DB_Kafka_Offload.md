## About

This connector can be used to offload files containing alarms, trending data, information events, etc. generated by DataMiner to a Kafka broker.

> [!NOTE]
> This integration is not ready as authenticating to a Kafka 

## Key Features

- **Offload control**: Offload can be toggled on/off at any time for any topic.

- **Modular topic separation**: Customizable offload into separate Kafka topics for each information type.

- **Optimization and configuration**: Configurable settings like offload intervals, retry mechanisms and storage limit.

## Use Cases

### Use Cases

#### Data usability with different environments

**Challenge**: Using the offloaded data with all sorts of environments.

**Solution**: Kafka offers easy integration with pipelines, data lakes, indexing engines, databases, etc.

**Benefit**: Allows for a wider variety of integrations with systems running DataMiner.
